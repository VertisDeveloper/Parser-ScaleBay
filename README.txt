Парсер (parser_scalebay.py) создан для сбора данных товаров  с сайта - https://scalebay.ru. 
Собирает такие данные как - (Название, Ссылки на картинки, Номер лота, Блиц цена, Марка, Производитель, Масштаб, Материал, Артикул, Состояние, Описание, Колличество товара, Ссылка на продавца, Ник продавца).
Есть 2 режима работы:
1) Парсинг по ссылкам продавцов - links_sellers.txt. Сохраняет в файл формата CSV
2) Парсинг всего ассортимента сайта. Сохраняет в файл формата XLSX

Оба режима ждут setting файл  - final_config.json.

1. Для создания config.json файла запускаем parser_categories_and_manufacturers.py. Внутри файла на строке 137 задаем время парсинга в параметре hour=ваше число от 0 до 23, minute=ваше число от 0 до 59. Для этого скрипта требуется файлы - соответствия.csv (этот файл нужен был чтобы изменять производителей продавцов на свои) с разделителем - ;, файл list_all_manufacturers.py (этот файл нужен для сравнения всех производителей с производителями продавцов), файл categories_dict.py (этот файл нужен для формирования ссылок на категории у каждого продавца чтобы ваыбрать необходимые).

ВНИМАНИЕ! Основной парсер ожидает final_config.json подразумевается что клиент сам его создаст за основу берется config.json.

2. Основной парсер (parser_scalebay.py) ожидает файл final_config.json. В нем должна содержатся следующая информация.
- В ключе parsing_all_site по умолчанию False. Если нужно парсить все товары сайта ScaleBay поставьте True
- В ключе 'parsing_time',нужно оставить 1 словарь(режим парсинга). Если выбрать  ключ 'everyday' и задать hour - число от 0 до 23  и minute - число от 0 до 59 - в этом случае парсинг будет ежедневно. Если выбрать ключ 'interval' и задать hour - число от 1 до 23 в этом случае парсинг каждые N-ое колличество часов.
- В ключе time_trade число, время до окончания торгов пример - 24
- В ключе markup число, наценка (проценты) пример  - 30
- В ключе price_delivery число, стоимость доставки (в рублях) пример - 300
- В ключе condition оставляете слова которые нужно парсить либо Новый либо Б/у либо оставить все как есть если нужно парсить оба состояния. пример - ["Новый", "Б/у"]
- В ключе categories нужно оставить ссылки на категории которые НУЖНО ПАРСИТЬ должен быть список - пример
["https://scalebay.ru/category/masshtabnye-modeli_otechestvennye-avtomobili", "https://scalebay.ru/category/sbornye-modeli_avtomobili"]
- В ключе unnecessary_manufacturers добавить только тех производителей которые НЕ НУЖНО ПАРСИТЬ.
- В ключе all_manufacturers содержатся все производители продавцов которые были изменены на нужные из файла соответствия.csv. Этот словарь не трогать, он остается как есть.
